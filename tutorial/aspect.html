How to wrap a request call with a particular functionality

Aspect is the class which provides a possibility to add some kind of functionality before the RouteAction (the body of an endpoint) is called and then add some functionality after the action is called. Aspects are ordered to the ActionChain according to their Order number.

The interface ActionChain contains a method InternalResponse<Object> proceed(InternalRequest<Object> request, InternalResponse<Object> response) that is used to continue in the aspect's chain or in the case when the currently processed aspect is the last, so then to process a body of the particular route.

There are several available aspects:

Aspect Name Description Default
SerializationAspect The aspect transform body of the request or response to/from an entity according to provided MediaType YES
