How to Create Controller and Route

Controllers are configuration classes which provides all information about routes that have something similar to each other. For example resource Joke should have one controller configuration with all operations which are available to do with this resource. There is one abstract class TypedControllerConfiguration that makes a route creation very easy because contains various combinations of parameters to makes a work with it as convenient as possible. Its usage is shown below.

There are several ways to configure the body of the route:

  • using a lambda expression
  • using a method reference
  • using an implementation of the interface BiConsumer<Request<REQ>, Response<RESP>>

Maybe someone may find strict declaring of Request/Response types as a verbose and annoying activity, but this is the biggest advantage of JoyRest. Since this strict defining types allows to JoyRest resolve Writes/Readers during an application bootstrapping and not during a Request processing. This is a trade-off between an additional written code and a performance impact.

JoyRest chose a better performance !!

Lambda expression example


public class JokeController extends TypedControllerConfiguration {

    @Inject
    private JokeService service;

    @Override
    protected void configure() {
        get((req, resp) -> {
            List<Joke> jokes = service.getAll();
            resp.entity(jokes);
        }, RespList(Joke.class)).produces(JSON, XML);
    }
}
